# Internationalization

WEB-11 adds a lightweight Turkish/English i18n foundation for the code-first web app.

## Architecture

- Turkish (`tr`) is the default locale and keeps the canonical routes without a `/tr` prefix.
- English (`en`) uses `/en` prefixed routes.
- Middleware resolves locale in this order:
  1. Explicit route prefix such as `/en`.
  2. `otoyali_locale` cookie.
  3. `Accept-Language` on first visit.
  4. Turkish fallback.
- `/tr/*` redirects back to the canonical unprefixed Turkish route.
- `/en/*` is internally rewritten to the existing App Router pages after converting safe English route aliases back to Turkish/internal paths.
- `/akis` remains a legacy redirect to `/video`; it is not user-facing and is excluded from sitemap.

The implementation is intentionally small:

- `src/i18n/config.ts` stores locale config, route aliases, cookie/header names, and path helpers.
- `src/i18n/dictionaries/tr.ts` and `src/i18n/dictionaries/en.ts` store product UI dictionaries.
- `src/i18n/get-dictionary.ts` provides server/client dictionary access and interpolation.
- `src/i18n/server.ts` reads the request locale from middleware headers/cookies.
- `src/i18n/client.tsx` exposes `I18nProvider` and `useI18n`.
- `src/middleware.ts` handles locale resolution, redirects, and rewrites.
- `src/lib/format.ts` centralizes locale-aware currency, number, date, mileage, city, and enum labels.

No external i18n dependency was added.

## Route Rules

Turkish canonical examples:

- `/`
- `/search`
- `/video`
- `/listing/[id]`
- `/sell`
- `/login`
- `/profile`
- `/my-listings`
- `/ikinci-el-araba`
- `/marka/[slug]`
- `/sehir/[slug]`

English public examples:

- `/en`
- `/en/search`
- `/en/video`
- `/en/listing/[id]`
- `/en/sell`
- `/en/login`
- `/en/profile`
- `/en/used-cars`
- `/en/make/[slug]`
- `/en/city/[slug]`

Do not add `/tr` links to production UI. Use `localizePath(href, locale)` for internal links.

## Translation Coverage

Current coverage focuses on MVP user-facing surfaces:

- Shell navigation, footer, mobile nav, language switcher.
- Home page, marketplace cards, search filters, active chips, sorting, saved search prompts.
- Listing details, seller CTA, share/report flows, price/spec/date/mileage labels.
- Login and OTP.
- Sell entry page and key wizard labels/options.
- Favorites, profile, settings, my listings entry states and localized redirects.
- OTOYALI Video entry and cards.
- Legal/trust pages and SEO route metadata basics.
- Admin shell/login navigation only; deep moderation table copy remains mostly Turkish for now.

User-generated listing titles and descriptions are not translated. Database catalog values are not destructively changed.

## SEO

- `src/app/sitemap.ts` emits Turkish canonical public routes and English `/en` alternates for safe public routes.
- `src/app/robots.ts` keeps public Turkish and English routes crawlable while blocking private, admin, auth, API, and debug routes.
- Admin, auth callback, debug, and `/akis` are excluded from sitemap.
- Future per-page metadata can use `buildSeoMetadata` alternates, but WEB-11 keeps the change minimal.

## Future Locales

`de`, `ar`, and `ru` are reserved in config for future expansion, but they are not active. Arabic will require RTL visual QA before activation.

## Developer Rules

- Use `getRequestLocale()` and `getDictionary(locale)` in server components.
- Use `useI18n()` in client components.
- Use `localizePath()` for internal links and preserved `next` redirects.
- Use format helpers from `src/lib/format.ts`; avoid hardcoded `tr-TR` in new UI.
- Keep query parameter keys ASCII and stable.
- Do not translate UGC or rewrite stored DB values.
- Do not add third-party i18n libraries unless route complexity grows enough to justify it.
